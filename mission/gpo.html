🏢 Gestion des OU - Intégration Utilisateur

Procédure Novares - Suite à Ticket Freshdesk

Référentiel E4 - A.2.1.1

📋 Contexte de la procédure

Cette procédure décrit l'intégration d'une personne dans une Unité Organisationnelle (OU) Active Directory suite à la réception d'un ticket Freshdesk dans l'environnement Novares.

🏢 Site de Lens

Centre d'expertise

🏭 Site de Libercourt

Usine

🏭 Site de Villers-Bretonneux

Usine

🎯 Déclenchement : Cette procédure est initiée lors de la réception d'un ticket Freshdesk pour l'intégration d'un nouvel utilisateur.

⚙️ Procédure d'intégration dans une OU

1

Réception et analyse du ticket Freshdesk

Analyser le ticket reçu et identifier les informations nécessaires pour l'intégration.

📧 Exemple de ticket Freshdesk reçu
🎫 Ticket #FR-2024-0856 - Demande d'accès OU

Demandeur : Marie Dupont (Manager RH)

Utilisateur : Jean Martin

Service : Production - Lens

OU cible : OU=Production,OU=Lens,DC=novares,DC=local

💡 Information : Vérifier que toutes les informations requises sont présentes dans le ticket.
2

Vérification de l'utilisateur dans Active Directory

Rechercher l'utilisateur dans Active Directory pour vérifier son existence et ses paramètres actuels.

🔍 Recherche utilisateur dans Active Directory
PowerShell - Recherche utilisateur AD
PS C:\> Get-ADUser -Identity "jmartin" -Properties *
Utilisateur trouvé: Jean Martin (jmartin)
OU actuelle: OU=Temp,DC=novares,DC=local
Compte actif: True
🔍 Vérification : S'assurer que le compte utilisateur existe et est dans un état valide.
3

Déplacement de l'utilisateur vers la nouvelle OU

Effectuer le déplacement de l'utilisateur vers l'OU de destination spécifiée dans le ticket.

🔄 Déplacement de l'utilisateur vers la nouvelle OU
🔄 Migration : Le déplacement peut nécessiter un délai de propagation pouvant aller jusqu’à une heure. Ce délai s’explique par le passage obligatoire via la passerelle avant l’acheminement vers le site de Villers.
4

Vérification et validation du déplacement

Confirmer que l'utilisateur a bien été déplacé et que tous ses attributs sont corrects.

✅ Vérification du déplacement réussi

Points de contrôle :

✅ Utilisateur présent dans la nouvelle OU
✅ Permissions héritées correctement
✅ Accès aux ressources fonctionnel
✅ Stratégies de groupe appliquées
🎯 Validation : Tester l'accès de l'utilisateur aux ressources de sa nouvelle OU.
5

Application des stratégies de groupe

Vérifier que les stratégies de groupe (GPO) appropriées sont appliquées à l'utilisateur dans sa nouvelle OU.

🛡️ Sécurité : Les stratégies de groupe définissent les permissions et restrictions de l'utilisateur.
6

Notification et clôture du ticket

Informer le demandeur de la réalisation de la demande et clôturer le ticket Freshdesk.

✅ Réponse et fermeture du ticket
✅ Ticket #SR-309998 - RÉSOLU

Statut : Résolu

Action réalisée : Utilisateur Jean Martin déplacé vers OU=Production,OU=Lens

Date de résolution : 18/09/2024 - 14:30

Technicien : Service IT Novares

📝 Documentation : Conserver une trace de l'intervention pour l'historique.
7

Suivi post-déplacement

Effectuer un suivi dans les 24-48h pour s'assurer que l'utilisateur n'a pas de problème d'accès.

📊 Monitoring : Surveiller les logs d'authentification pour détecter d'éventuels problèmes.

🔍 Points de contrôle essentiels

✅ Ticket analysé : Toutes les informations vérifiées
✅ Utilisateur vérifié : Compte valide dans AD
✅ Déplacement effectué : OU mise à jour
✅ Permissions validées : Accès fonctionnel
✅ Ticket clôturé : Demandeur informé
