SISR — Épreuve E6

PPE 2 – Infrastructure Web & Haute Disponibilité

Déploiement d'une architecture 3-tiers résiliente pour le site de réservation CICAR El Hierro.

1. Présentation du besoin

L'agence CICAR nécessite une plateforme de réservation en ligne disponible 24h/24. L'objectif de ce PPE est de supprimer tout "Point Unique de Défaillance" (SPOF) au niveau de la couche web en mettant en œuvre un répartiteur de charge (Load Balancer) et une grappe de serveurs redondants.

Périmètre

  • Virtualisation VMware ESXi
  • Zone DMZ (Sécurité externe)
  • Zone LAN (Données internes)

Technologies

  • HAProxy (Reverse Proxy)
  • Apache2 & PHP 8.2
  • MariaDB (Master-Slave)

2. Architecture Technique

L'infrastructure repose sur la segmentation réseau définie sur le pare-feu Stormshield SNS 310 :

Configuration du Reverse Proxy (SRV-EL-PROXY)

Le serveur HAProxy reçoit les requêtes HTTPS et les distribue selon l'algorithme Round-Robin.

# /etc/haproxy/haproxy.cfg
backend web-backend
    balance roundrobin
    option httpchk GET /health.php
    server WEB-01 192.168.54.15:80 check
    server WEB-02 192.168.54.16:80 check

Réplication de la Base de Données

Pour garantir l'intégrité des réservations, MariaDB est configuré en réplication pour assurer la persistance des données sur les deux nœuds.
