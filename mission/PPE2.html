1. Contexte

L’entreprise CICAR possède plusieurs agences réparties sur différentes îles. Le service de réservation dépend d’une infrastructure réseau insuffisamment fiable, provoquant des interruptions de service.

2. Objectifs

  • Assurer la continuité de service
  • Améliorer la disponibilité des applications
  • Sécuriser les échanges inter-sites

3. Architecture proposée

  • Architecture 3-tiers (LAN / DMZ / WAN)
  • Load balancing avec HAProxy
  • Serveurs Web Apache
  • Pare-feu Stormshield
  • VPN IPsec site à site

4. Mise en œuvre

  1. Création du réseau et des VLAN
  2. Configuration des pare-feux
  3. Mise en place du VPN IPsec
  4. Installation des serveurs Apache
  5. Configuration HAProxy (load balancing)
  6. Tests de basculement et disponibilité

5. Résultats

  • Service hautement disponible
  • Répartition de charge efficace
  • Sécurité renforcée des communications
