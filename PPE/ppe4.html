La demande
Présentation

CICAR est une société de location automobile implantée aux Îles Canaries depuis 2012. L'entreprise possède une flotte de 65 véhicules répartis dans plusieurs agences. Dans le cadre de son expansion, l'agence d'El Hierro (Valverde) a été équipée d'une infrastructure réseau complète.

Les îles étant assez disséminées dans l'archipel, l'administrateur du siège de Tenerife souhaite un accès distant sécurisé sur tous les équipements paramétrables et serveurs de chaque agence.

Objectifs de la PPE4
  • Mise en place de tous les accès distants nécessaires à l'administrateur (centralisation sous MobaXterm)
  • Mise en place et test de l'authentification 2FA sur le SNS (Google Authenticator)
  • Gestion des flux dans les pare-feux pour permettre les accès distants
  • Installation du logiciel d'inventaire (GLPI_Agent) et de helpdesk (GLPI)
  • Test des remontées d'équipements et des tickets d'incidents
  • Rédaction d'une procédure utilisateur pour la plate-forme helpdesk
  • Mise à jour des tableaux de connexion et des schémas

🖥️ 1. Accès distant centralisé — MobaXterm

L'administrateur du siège utilise MobaXterm pour centraliser l'ensemble des connexions distantes vers les équipements de chaque agence. Via le tunnel VPN IPsec établi entre le Stormshield de l'agence El Hierro et le pare-feu du siège, MobaXterm permet :

  • Connexions SSH vers toutes les VMs Linux (WEB-01, WEB-02, MAIL, PROXY)
  • Connexions RDP vers le serveur Windows Server 2019 (AD)
  • Accès HTTPS à l'interface d'administration du Stormshield
📌 Appui à l'administrateur du siège : Recensement des connexions et ouverture des flux dans les pare-feux de chaque île.

🔐 2. Double authentification 2FA (Google Authenticator)

Pour renforcer la sécurité des accès administratifs, l'authentification à deux facteurs a été activée sur le pare-feu Stormshield SNS 310. La méthode retenue est le One Time Password (OTP) avec l'extension Google Authenticator.

# Configuration de la 2FA sur Stormshield SNS 310
# Étape 1 : Activer l'authentification multi-facteurs
Administration → Utilisateurs → admin → Activer "Double authentification"

# Étape 2 : Scanner le QR code avec Google Authenticator
Configuration → Sécurité → MFA → Générer un secret OTP

# Étape 3 : Tester la connexion
https://192.168.54.254 → saisir login/mdp → saisir code OTP

🌊 3. Gestion des flux (règles de filtrage)

Pour permettre les accès distants depuis le siège vers l'agence El Hierro via le tunnel VPN IPsec, les règles de filtrage suivantes ont été configurées sur le Stormshield SNS 310 :

📌 Précision : Il s'agit uniquement de règles de filtrage (firewall rules). Le NAT n'est pas utilisé ici car les accès se font via le tunnel VPN IPsec entre le siège et l'agence.
📦 4. GLPI — Inventaire et Helpdesk

GLPI est installé sur un serveur Debian au siège de Tenerife. Il est accessible depuis toutes les agences via l'URL https://glpi.el-cicars.es. Les fonctionnalités déployées :

  • GLPI_Agent installé sur tous les postes et serveurs pour l'inventaire automatique (matériel, logiciels, mises à jour)
  • Helpdesk : gestion centralisée des tickets d'incidents remontés par les chefs d'agence
  • Authentification LDAP : intégration à l'Active Directory el-cicars.es

🧪 5. Tests des remontées d'équipements et tickets d'incidents

Des tests ont été réalisés pour valider le bon fonctionnement de la solution :

  • Test inventaire : Installation d'un nouveau poste → remontée automatique dans GLPI en moins de 30 minutes
  • Test ticket : Création d'un ticket depuis l'agence → notification reçue par l'administrateur du siège
  • Test workflow : Traitement du ticket, changement de statut, clôture avec commentaire
  • Test notification : Email automatique envoyé au demandeur lors de la prise en charge

📝 6. Procédure de création de ticket (Self-Service)

📌 Procédure utilisateur — Création d'un ticket d'incident sur GLPI

1
Ouvrir un navigateur et se rendre sur https://glpi.el-cicars.es
2
S'authentifier avec les identifiants du domaine el-cicars.es (login/mot de passe Windows)
3
Dans le menu, cliquer sur « Assistance » puis « Créer un ticket »
4
Sélectionner la catégorie de l'incident (Matériel, Logiciel, Réseau, Impression, Autre)
5
Évaluer l'urgence de « Très basse » à « Très haute » (selon l'impact sur l'activité)
6
Associer le matériel concerné (ex: poste EL-PC-01, imprimante EL-IMP-01) si applicable
7
Saisir un titre clair et une description détaillée du problème (avec captures d'écran si possible)
8
Cliquer sur « Soumettre la demande »
9
Un email de confirmation est reçu. Le ticket sera traité par l'administrateur du siège.
